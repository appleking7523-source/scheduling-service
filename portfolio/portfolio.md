# 일정 관리 서비스 포트폴리오 문서

## 프로젝트 개요

- 프로젝트명: What To Do
- 유형: 개인 일정 및 할 일 관리 서비스
- 산출물: 반응형 HTML/CSS 기반 웹 프로토타입
- 핵심 화면: 온보딩, 로그인/회원가입, 홈, 일정, 할 일, 프로필
- 목표 사용자: 학업, 개인 일정, 과제를 동시에 관리해야 하는 학생 및 자기관리 초보 사용자

What To Do는 오늘 해야 할 일과 시간표를 한 화면에서 확인하고, 진행률을 통해 하루의 상태를 빠르게 파악하도록 설계한 일정 관리 서비스이다. 사용자가 복잡한 기능보다 "오늘 무엇을 해야 하는지"에 집중할 수 있도록 홈 화면, 캘린더, 할 일 목록, 진행률 표시를 중심으로 구성했다.

## 1. 문제정의

### 사용자 문제

사용자는 일정과 할 일을 서로 다른 앱, 메모, 캘린더에 흩어 기록하면서 다음과 같은 불편을 겪는다.

- 오늘 해야 할 일과 시간 기반 일정이 분리되어 우선순위를 잡기 어렵다.
- 완료한 일의 양을 직관적으로 확인하기 어려워 성취감이 약하다.
- 일정 관리 도구가 복잡하면 입력 자체를 미루게 된다.
- 모바일에서는 빠른 확인이 중요하지만, 데스크톱에서는 하루 계획을 넓게 보고 싶다.

### 핵심 문제 문장

학생 사용자는 과제, 수업, 모임처럼 성격이 다른 일을 하루 단위로 정리해야 하지만, 일정과 할 일이 분리되어 있어 오늘의 진행 상황과 다음 행동을 빠르게 판단하기 어렵다.

### 해결 방향

- 홈 화면에서 오늘의 할 일, 일정, 진행률을 함께 제공한다.
- 캘린더와 리스트를 결합해 시간 기반 일정과 체크리스트를 동시에 탐색하게 한다.
- 하단 내비게이션과 플로팅 추가 버튼으로 모바일 사용 흐름을 단순화한다.
- 데스크톱에서는 캘린더와 보조 정보를 나란히 배치해 정보 탐색 효율을 높인다.

## 2. 가설

### 가설 1: 오늘 중심의 요약 화면은 진입 부담을 줄인다.

사용자가 앱을 열었을 때 오늘의 할 일, 일정, 진행률을 바로 확인할 수 있다면 별도의 탐색 없이 다음 행동을 결정할 수 있다.

### 가설 2: 진행률 시각화는 완료 행동을 강화한다.

체크 완료 수와 퍼센트 진행률을 함께 보여주면 사용자는 남은 작업량을 빠르게 이해하고, 완료 행동에서 성취감을 얻을 수 있다.

### 가설 3: 일정과 할 일을 분리하되 연결하면 관리 효율이 높아진다.

일정 화면은 날짜와 시간 흐름에 집중하고, 할 일 화면은 상태와 필터에 집중하게 구성하면 사용자는 목적에 맞게 화면을 선택하면서도 홈에서 전체 맥락을 잃지 않는다.

### 가설 4: 반응형 레이아웃은 같은 서비스의 사용 맥락을 넓힌다.

모바일에서는 빠른 확인과 추가, 데스크톱에서는 캘린더와 목록의 동시 확인을 제공하면 사용 장소가 바뀌어도 동일한 서비스 경험을 유지할 수 있다.

## 3. 디자인 컨셉

### 컨셉 키워드

- Today First: 하루 계획을 가장 먼저 보여준다.
- Light Productivity: 부담스럽지 않은 생산성 도구를 지향한다.
- Clear Status: 완료, 진행, 남은 일을 시각적으로 분명하게 보여준다.
- Responsive Routine: 모바일과 데스크톱 사용 흐름을 모두 고려한다.

### 비주얼 방향

- 메인 컬러는 신뢰감 있는 블루를 사용해 일정 관리 서비스의 안정감을 표현한다.
- 흰색 카드와 옅은 회색 배경으로 정보 영역을 분리해 가독성을 높인다.
- 체크 아이콘, 캘린더, 일정 썸네일, 진행률 바를 통해 텍스트 의존도를 낮춘다.
- 라운드와 그림자를 적당히 사용해 부드럽지만 도구다운 인상을 유지한다.

### UX 원칙

- 첫 화면에서 오늘의 상태를 파악할 수 있어야 한다.
- 추가, 확인, 이동 같은 반복 행동은 한 번에 찾을 수 있어야 한다.
- 화면마다 역할을 명확히 나누되, 내비게이션 구조는 일관되게 유지한다.
- 모바일과 데스크톱의 정보 밀도를 다르게 설계한다.

## 4. 화면 소개

### 온보딩

서비스의 목적을 짧은 문장과 일러스트로 안내한다. 사용자는 일정 정리와 과제 관리라는 핵심 가치를 먼저 이해한 뒤 로그인 화면으로 이동한다.

### 로그인 및 회원가입

이메일 기반 로그인과 소셜 로그인 옵션을 제공한다. 데스크톱에서는 좌측에 서비스 화면 이미지를 배치해 제품 맥락을 먼저 보여주고, 우측에서 입력을 진행하도록 구성했다.

### 홈

오늘 할 일, 오늘 일정, 하루 진행률, D-day, 캘린더 요약을 한 화면에 배치했다. 사용자는 앱 진입 직후 오늘의 핵심 정보를 확인하고 일정 또는 할 일 상세 화면으로 이동할 수 있다.

### 일정

월간 캘린더와 선택 날짜의 일정을 함께 보여준다. 모바일에서는 상단 캘린더와 하단 일정 리스트 구조를 사용하고, 데스크톱에서는 미니 캘린더와 오늘 할 일을 보조 영역으로 배치했다.

### 할 일

검색, 기간 탭, 상태 필터, 정렬, 완료 체크를 제공한다. 진행 중인 할 일과 완료한 일을 구분해 사용자가 남은 과제를 빠르게 확인할 수 있다.

### 프로필

사용자 정보, 하루 진행률, 완료한 할 일 수, 공지사항, 문의사항, 설정, 로그아웃을 배치했다. 개인 성과와 계정 관련 기능을 한 곳에 모아 서비스 사용을 마무리하는 화면으로 구성했다.

## 6. 개선 방향

- 실제 데이터 저장 기능을 추가해 할 일과 일정을 사용자별로 관리한다.
- 날짜 선택, 일정 추가, 할 일 필터가 실제 데이터와 연결되도록 JavaScript 로직을 확장한다.
- 깨진 한글 텍스트를 정리하고 접근성 레이블을 개선한다.
- 일정 반복, 알림, 우선순위 기능을 추가해 실사용성을 높인다.
